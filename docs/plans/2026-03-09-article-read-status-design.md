# 文章已读/未读状态功能设计

## 概述

为 RSS Reader 添加文章已读/未读状态功能，支持用户追踪阅读进度。

## 需求

1. 点击文章时自动标记已读
2. 批量操作：全部标记已读、按订阅源标记、按分类标记
3. 未读筛选：可切换"只看未读"或"全部"

## 技术设计

### 数据模型

Article 模型已有 IsRead bool 字段，无需修改。

### 后端 API

#### 1. 标记单篇文章已读

PATCH /api/articles/:id/read

#### 2. 批量标记已读

POST /api/articles/mark-all-read

#### 3. 获取未读数量

GET /api/articles/unread-count

#### 4. 修改文章列表 API

添加 is_read 查询参数

### 前端改动

1. 未读文章：加粗标题 + 左侧蓝色标记
2. 顶部工具栏：新增"只看未读"开关、"全部标记已读"按钮
3. 侧边栏：订阅源/分类旁显示未读数量
